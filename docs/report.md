# 基于 Dry Bean Dataset 的多分类机器学习工程实验报告

## 摘要

本文基于教师提前划分好的 Dry Bean Dataset 脏数据训练集、验证集和测试集，完成一个包含数据分析、数据清洗、特征工程、多算法实验、系统展示和课程总结的机器学习工程项目。数据来源于干豆图像的形态学特征，任务目标是根据面积、周长、轴长、圆度、紧致度等 16 个数值特征预测干豆类别。项目实现了 RandomForest、MLP 和 XGBoost 三种多分类算法，其中 XGBoost 为课堂外补充算法。实验从测试集准确率、宏平均 F1、loss 曲线、推理速度、鲁棒性、过拟合差距和混淆矩阵等维度进行对比，并自动生成 GitHub Pages 风格的静态展示页面。

## 1. 数据分析

Dry Bean Dataset 是一个典型的结构化表格多分类数据集。每个样本对应一粒干豆，特征由图像处理算法提取，主要刻画豆子的大小、轮廓、长短轴比例、圆度、凸包面积和形状因子等。目标列 `Class` 表示干豆品种。

本项目使用的数据并非完全干净，训练集原始样本数为 9527，清洗后为 9506，测试集样本数为 2737。训练集中包含 16 个数值特征和 7 个真实类别。主观观察和程序统计发现以下数据污染：

- 训练集中存在 21 行重复样本，若不删除会增强模型的记忆倾向。
- 原始训练集存在 741 个缺失单元格。
- 数值列中存在 460 个非法或不可直接解析的数值单元格，需要强制转换后再填补。
- 类别标签存在大小写混乱、首尾空格和字符污染，例如 `D3RMAS0N`、`S3K3R`、`H0R0Z`、`B0MBAY`。
- 类别分布不完全均衡，其中 `DERMASON` 和 `SIRA` 样本较多，`BOMBAY` 样本较少。
- 相关性热力图显示面积、周长、凸包面积、等效直径等特征之间存在较强相关，说明数据有一定冗余。

## 2. 数据处理与特征工程

数据处理模块主要位于 `src/data_processing.py`。处理流程如下：

1. 列名标准化：去除列名前后的空白字符，避免后续访问失败。
2. 数值特征清洗：对除 `Class` 外的所有列执行数值转换，逗号会先移除，无法转换的内容记为缺失值。
3. 重复样本删除：使用 `drop_duplicates` 删除完全重复行。
4. 缺失值填补：对每个数值列使用该列中位数填补。中位数比均值更不容易被极端值影响。
5. 标签清洗：将类别标签统一转为大写，先把 `0/3/1/5` 等常见字符污染还原为 `O/E/I/S`，再去除非字母字符。
6. 标签编码：使用 `LabelEncoder` 将类别转为整数，以适配 XGBoost 等模型。
7. 特征标准化：使用 `StandardScaler` 对训练集拟合均值和方差，并复用到测试集。这样可以避免测试集信息泄漏，同时提升 MLP 的训练稳定性。
8. 噪声注入：鲁棒性实验中实现了数值特征高斯噪声和标签翻转噪声两类扰动。

这套流程覆盖了数据清洗、格式统一、缺失值处理、标签修复、标准化和鲁棒性增强实验所需的数据变换。

## 3. 多算法实验分析

### 3.1 算法说明

RandomForest 是基于多棵决策树投票的集成算法，优点是对特征尺度不敏感、可解释性较好、训练稳定；缺点是模型文件较大，推理速度相对慢。

MLP 是多层感知机神经网络，本项目使用一个隐藏层的 `MLPClassifier`。它对标准化较敏感，但推理速度快，且可以输出 loss 曲线便于观察优化过程。

XGBoost 是梯度提升树模型，属于课堂外扩展算法。它在结构化表格数据上通常表现强，能建模非线性关系，本项目使用 `mlogloss` 作为多分类评估指标。

### 3.2 主实验结果

| 模型 | 训练准确率 | 测试准确率 | 过拟合差距 | 推理耗时 ms/样本 | Macro-F1 |
|---|---:|---:|---:|---:|---:|
| RandomForest | 1.0000 | 0.9236 | 0.0764 | 0.0358 | 0.9330 |
| MLP | 0.9458 | 0.9266 | 0.0193 | 0.0016 | 0.9352 |
| XGBoost | 1.0000 | 0.9207 | 0.0793 | 0.0049 | 0.9334 |

从测试集准确率看，MLP 最高，为 0.9266；RandomForest 为 0.9236；XGBoost 为 0.9207。三者差距不大，说明清洗后的特征已经能较好支持多分类任务。

从宏平均 F1 看，MLP 仍最高，为 0.9352。由于数据存在类别不均衡，宏平均 F1 比单纯准确率更能反映小类别上的整体表现。

### 3.3 Loss 曲线对比

MLP 和 XGBoost 都能输出训练过程曲线，图表保存为：

- `experiments/full_run/MLP_loss.png`
- `experiments/full_run/XGBoost_loss.png`
- `experiments/full_run/loss_curves_compare.png`

MLP 在 400 次迭代时仍出现未完全收敛提示，说明继续增大迭代次数、调整学习率或加入更强正则化可能仍有提升空间。XGBoost 的多分类 logloss 曲线用于观察训练集和测试集之间的差距，便于判断是否过拟合。

### 3.4 推理速度对比

MLP 平均单样本推理耗时约 0.0016 ms，速度最快；XGBoost 约 0.0049 ms；RandomForest 约 0.0358 ms。对于需要低延迟预测的场景，MLP 更适合部署；如果更关注树模型的鲁棒性和可解释性，可以选择 RandomForest 或 XGBoost。

### 3.5 鲁棒性对比

项目在训练数据中加入两类噪声：

- `gaussian_feature`：对数值特征加入标准差为 0.01、0.05、0.10 的高斯噪声。
- `label_flip`：随机翻转 1%、5%、10% 的训练标签。

部分结果如下：

| 模型 | 噪声类型 | 强度 | 测试准确率 |
|---|---|---:|---:|
| RandomForest | gaussian_feature | 0.00 | 0.9236 |
| MLP | gaussian_feature | 0.00 | 0.9266 |
| XGBoost | gaussian_feature | 0.00 | 0.9207 |
| RandomForest | gaussian_feature | 0.10 | 0.9035 |
| MLP | gaussian_feature | 0.10 | 0.9017 |
| XGBoost | gaussian_feature | 0.10 | 0.9006 |
| RandomForest | label_flip | 0.10 | 0.9200 |
| MLP | label_flip | 0.10 | 0.9291 |
| XGBoost | label_flip | 0.10 | 0.9174 |

高斯特征噪声会让三种模型都有明显下降，说明形态特征的精度对分类结果有较大影响。标签翻转实验中，由于翻转比例不高且模型仍能从大量正确样本中学习，下降幅度相对小；MLP 在该设置下仍保持较好测试精度。

### 3.6 过拟合分析

RandomForest 和 XGBoost 的训练准确率均为 1.0000，但测试准确率约为 0.92，过拟合差距分别为 0.0764 和 0.0793。这说明树模型对训练集拟合能力很强，但需要通过限制树深、减少树数量、增大正则化、早停等方式控制泛化误差。

MLP 的训练准确率为 0.9458，测试准确率为 0.9266，差距仅 0.0193，泛化更稳定。结合推理速度和测试集表现，本项目最终推荐 MLP 作为当前默认模型。

## 4. 系统展示与工程集成

项目已经按照工程文件夹架构组织，核心模块包括：

- `src/data_processing.py`：数据加载、清洗、标准化和噪声注入。
- `src/models/core.py`：模型定义、训练、保存、加载和基础评估。
- `src/evaluate.py`：推理速度、分类报告和 loss 曲线绘制。
- `src/run_experiments.py`：统一命令入口，负责串联 EDA、训练、测试、鲁棒性实验、图表生成和网页生成。
- `docs/index.html`：静态系统展示界面。

统一运行命令：

```bash
python -m src.run_experiments --data-dir .. --out experiments/full_run --site-dir docs
```

算法运行阶段不会弹出 UI，所有输出均保存到文件。系统展示页面包含数据描述、数据处理方法、算法实现说明、精度表、loss 曲线、推理速度、鲁棒性曲线和混淆矩阵，适合作为 GitHub Pages 展示。

## 5. 课程总结

通过本课程和本次期末项目，我对机器学习工程流程有了更完整的理解。以前做实验时容易只关注模型训练和最终准确率，但本项目让我意识到，数据质量、清洗策略、实验设计、评估维度和展示方式同样重要。

在数据处理方面，我学习到真实数据中常见的缺失值、重复样本、格式污染和标签污染会直接影响模型结果。例如本项目一开始如果只删除非字母字符，会把 `D3RMAS0N` 错误处理成新类别，导致 7 类任务变成 11 类任务。这个问题说明数据清洗不能只写代码，还要结合业务含义检查结果是否合理。

在算法实验方面，我对比了神经网络、随机森林和 XGBoost。三种算法都能达到较高准确率，但它们在速度、过拟合和鲁棒性上有明显差异。单一准确率不能完整评价模型，宏平均 F1、推理延迟、训练/测试差距和噪声扰动后的性能变化都很有参考价值。

在工程实现方面，我学习到项目需要有清晰目录结构和统一入口，不能只散落在 Notebook 中。通过 `python -m src.run_experiments` 一条命令自动生成结果，可以让实验更容易复现，也方便教师检查。网页展示和 README 则让项目更像一个完整作品，而不是零散作业文件。

对课程的建议是：希望后续课程能增加更多真实脏数据案例、模型部署案例和项目展示规范，让学生更早形成工程化意识。同时也可以在课堂中加入更多课堂外算法介绍，如 XGBoost、LightGBM、CatBoost 等，让学生理解传统机器学习和现代集成方法在表格数据上的优势。

## 6. 结论

本项目完成了期末考核要求的全流程机器学习工程。实验结果显示，在当前数据处理方案下，MLP 以 0.9266 的测试集准确率和 0.9352 的 Macro-F1 取得最佳综合表现，同时推理速度最快、过拟合差距最小。RandomForest 和 XGBoost 表现接近，但训练集准确率过高，存在更明显过拟合。
